# Version A (Manual Version)

This folder contains the layouts coded entirely manually from scratch without any AI tools or assistive software. The objective was to practice native element stacking and build core document flows strictly using pure HTML structure.

## Folder Contents & Implementation

This folder includes two foundational web layouts implemented with basic, sequential HTML elements:

1. **Survey Form (`survey-form.html`)**
   - Built using a classic top-to-bottom document flow where elements are arranged sequentially.
   - Uses native line breaks (`<br>`) and horizontal rules (`<hr>`) to separate fields and maintain a clean layout structure.
   - Contains all necessary interactive elements like text inputs, email types, numerical ranges, dropdown selectors, grouped radio buttons for single choices, and unique checkboxes for multi-selections.

2. **E-Commerce Product Page (`ecommerce-product.html`)**
   - Features a linear layout where content blocks stack naturally one after another down the page.
   - **Header Section:** Built with simple text blocks, headings, and character dividers (`|`) to create a functional navigation menu.
   - **Visual Elements:** Embeds a primary hero image of the luxury watch alongside inline thumbnail views sequentially beneath it using raw image tags (`<img>`).
   - **Product Specs & Forms:** Displays typography tags (`<h2>`, `<h1>`, `<h3>`) to map content hierarchy (pricing, discounts, and item description). Includes a basic select input for quantity tracking and a standard action button for cart submission.

## Strict Compliance Metrics
- **No CSS Styles:** Completely free of external stylesheets, embedded `<style>` containers, or inline style attributes. 
- **Authentic Manual Development:** The layout leans entirely on default browser rendering and basic block/inline flows, fulfilling the strict human-authored requirements of this assignment.
